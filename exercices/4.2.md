# Exercice 4.2 : modification de la structure d'une table

A partir de votre base de données de l'exercice [0.3.1](./0.3.1.md), apportez des modifications à la structure de vos tables pour intégrer les règles de gestion suivantes

1. L'identifiant d'une région est une clé primaire.
2. L'identifiant d'un pays est une clé primaire.
3. L'identifiant d'une localisation est une clé primaire.
4. La colonne région dans pays est une clé étrangère qui fait référence à l'identifiant d'une région dans la table région.
5. Le nom d'une région est unique.
6. Pays dans la table localisation est une clé étrangère.
7. Le nom d'une région doit avoir obligatoirement l'une des valeurs suivantes :
- Europe
- Amérique
- Asie
- Océanie
- Afrique
8. La taille du nom d'une région est désormais de 10 caractères au maximum
9. Le code postal d'une ville peut être nulle
10. Ajoutez des colonnes longitude et latitude dans la table localisation. Ces colonnes sont des nombres flottants avec une partie entière à 2 chiffres max.
11. Supprimez la colonne province de la table localisation.
12. Deux villes peuvent avoir le même nom.
